1. Introdução: O que são variáveis?

Na aula passada, vimos que dados (números, textos, booleanos) são processados pelos sistemas. Mas onde eles ficam guardados enquanto o programa está rodando?

A analogia das caixas:

Pense numa variável como uma caixa com uma etiqueta. Você guarda um valor dentro da caixa, e escreve um nome nela (a etiqueta) para encontrá-la depois.

  • A caixa: É o espaço na memória do computador.
  • O valor: É o dado em si (ex: "João", 1500).
  • A etiqueta: É o nome da variável (ex: nome_cliente, orcamento).

2. Como funciona na prática?

banda_favorita = "Oasis"

O sinal de = não significa "igual" (como na matemática). Na programação, ele significa RECEBE. Ou seja, a caixa com etiqueta banda_favorita acaba de receber o valor "Oasis".

Outro exemplo

Quando você acessa o site da SolarTech e vê "Olá, Maria!", o computador não gravou o texto "Maria" na tela para sempre.
Ele fez algo parecido com isso (em código Python):

nome_cliente = "Maria"
print("Olá, " + nome_cliente + "!")

O print é uma função que serve para exibir informações na tela.

3. Boas Práticas Básicas

Como na vida real, se você rotular todas as caixas da sua mudança apenas como "Coisas", você não achará nada depois. O mesmo vale para o código!

  • ❌ Ruim: x = 100 (O que é X?)
  • ✅ Bom: preco_kit_solar = 100 (Claro e objetivo)

Regras de Ouro: Nunca use espaços no nome das variáveis (use_underline). E nomes devem sempre fazer sentido dentro do seu contexto de negócio.
