Request & Response

Como a internet funciona?

↓

1. O Carteiro da Internet

Essa comunicação sempre segue uma regra rigorosa de duas vias, como os correios:

  • 1. Request (Requisição): É a carta que sai de você. Tem endereço, selo e sua pergunta ou dados. O Frontend sempre dá o primeiro passo.
  • 2. Response (Resposta): É o pacote que volta do Servidor. Trazendo a página de volta, dados no formato JSON ou, em certos casos, uma mensagem de erro indicando que a página não existe.

2. A Anatomia do Request (O que você envia)

O Backend não é vidente, por isso, ao acessar um site ou aplicativo, o seu celular despacha o Request contendo duas coisas muito importantes:

Ação Básica (Método): GET Endereço de Destino: www.site.com/produtos
  • O Método GET (Pegar) significa: "Por favor, me envie a lista de produtos".
  • O Método POST (Publicar/Enviar) é usado quando você está mandando informação nova. Exemplo: um formulário de cadastro sendo enviado. O Método POST leva dados de pagamento "escondidos" dentro do pacote.

3. O Response Status (A resposta do Servidor)

Quando o pacote de resposta volta, ele vem carimbado com um código numérico para o computador entender logo de cara se tudo deu certo ou se a coisa ficou feia.

  • 🟢 200 OK: "Tudo certo! Aqui está o que você pediu".
  • 🟡 404 Not Found: "Não encontrei essa página ou esse produto! Tem certeza do endereço?" (Erro histórico e super famoso da web).
  • 🔴 500 Server Error: "A culpa é nossa (do Backend). Deu pau no nosso código ou o Banco de Dados parou de funcionar e a conta de luz atrasou."
