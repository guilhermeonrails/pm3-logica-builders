O Idioma dos Sistemas

Métodos de API

Como a internet entende o que queremos fazer quando clicamos em um botão.

↓

1. O que é uma API?

Imagine que você fala português e o computador do banco fala japonês. Se vocês tentarem conversar diretamente, não vai dar certo. A API (Interface de Programação de Aplicações) é como um tradutor oficial que fica entre os dois.

De forma muito simples, APIs são pontes de comunicação entre sistemas. Quando você clica "Pagar com PIX" no iFood, o iFood não o seu banco. Em vez disso, o iFood usa uma API para mandar uma mensagem segura para o seu banco perguntando: "Ei, você pode aprovar esse pagamento?".

Isso acontece o tempo todo no seu dia a dia: ao fazer login com o Google em outro site, ver a previsão do tempo no celular ou buscar passagens aéreas.

2. Como os pedidos são feitos? (Ações)

Vimos na aula passada que tudo na internet funciona através de requisições (Requests). Mas como a API do servidor sabe o que você quer fazer?

Para manter as coisas organizadas, as APIs funcionam baseadas em "intenções". Sempre que fazemos um pedido, nós avisamos qual é a nossa intenção escolhendo um dos quatro grandes métodos: GET, POST, PUT ou DELETE. Pense neles como verbos em uma frase de ação.

3. O Método GET (Buscar)

O GET (do inglês "Pegar") é o método visualizador. Ele indica que você quer apenas BUSCAR ou LER informações, sem alterar absolutamente nada no servidor.

🍕 A Analogia do Restaurante: É o equivalente a você sentar na mesa e pedir o cardápio ao garçom. Você só quer ver o que tem lá.

Exemplo do Mundo Real: Ver produtos

Quando você abre a página inicial da Amazon, seu navegador faz um GET nos servidores pedindo a lista das ofertas do dia. O resultado esperado é que O Servidor retorne uma lista enorme contendo os nomes, fotos e preços para você simplesmente olhar.

4. O Método POST (Criar)

O POST (do inglês "Publicar/Enviar") é o método criador. Ele indica que você quer ENVIAR informações novas para que o servidor CRIE algo que não existia antes.

🍕 A Analogia do Restaurante: É o momento em que você chama o garçom e faz o seu pedido ("Quero uma pizza de calabresa"). Uma nova ordem de preparo acaba de nascer na cozinha!

Exemplo do Mundo Real: Criar Cadastro

Quando você preenche seu nome e senha para criar uma nova conta na Netflix, e aperta "Registrar", seu navegador envia um POST confidencial. O resultado esperado é o Servidor responder: "Recebido! Sua conta foi criada no nosso banco de dados com sucesso".

5. O Método PUT (Atualizar)

O PUT (do inglês "Colocar") é o método atualizador. Ele indica que algo já existe lá no banco de dados, mas você quer substituí-lo ou ATUALIZAR informações velhas.

🍕 A Analogia do Restaurante: A pizza já está sendo preparada, mas você lembra que é alérgico. Você chama o garçom e pede para "Tirar a cebola do pedido anterior". O cozinheiro altera a comanda e continua.

Exemplo do Mundo Real: Mudar o Endereço

Você se mudou e vai no Mercado Livre alterar o seu CEP de entrega na aba do perfil. Ao apertar salvar, o celular dispara um PUT. O resultado esperado é o Servidor substituir o seu CEP antigo pelo novo e confirmar: "Endereço atualizado".

6. O Método DELETE (Remover)

Como o nome sugere, o DELETE é o método destruidor. Ele possui uma única responsabilidade vital: REMOVER um dado específico definitivamente.

🍕 A Analogia do Restaurante: Você olhou o relógio, viu que demorou demais e avisa o garçom: "Pode cancelar meu pedido, por favor. Vou embora". A comanda é jogada no lixo e o prato cancelado.

Exemplo do Mundo Real: Deletar Conta

Depois de muita dor de cabeça, você clica naquele botão oculto de "Deletar Conta" do Facebook. O aplicativo envia uma mensagem silenciosa com o método DELETE. O resultado esperado é o servidor limpar todas as suas fotos e dados e confirmar: "Sua conta foi escluída para sempre".

7. Resumo Rápido (O Quadrado Mágico)

Lembre-se sempre dos verbos e você nunca errará a sua intenção técnica:

  • 👁️ GET → Buscar dados (Eu só quero ver!)
  • 🌱 POST → Criar dados (Estou gerando algo novo e inédito!)
  • ✏️ PUT → Atualizar dados (Já existe, mas vou corrigir!)
  • 🗑️ DELETE → Remover dados (Apague isso agora!)
